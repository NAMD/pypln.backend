Module Hierarchy

  • pypln: PyPLN -- Distributed text processing pipeline.

        • pypln.Monitors.webmonitor.monitor: Web app to monitor PyPLN cluster
        • pypln.Monitors.webmonitor.settings
    • pypln.apps
      • pypln.apps.cleaner: This script contains a number of cleanup functions which are intended to be run as a cron job with arguments indicating the type of cleanup to do.
      • pypln.apps.cluster
        • pypln.apps.cluster.cmanager: This executable is the manager of a pypln cluster. All tasks should be started through it and are monitorable/controllable through it.
        • pypln.apps.cluster.slavedriver: Created on 27/04/12 by fccoelho license: GPL V3 or Later
        • pypln.apps.cluster.test_cluster: Testing module for cmanager.py
      • pypln.apps.clusterer: Applies the clustering
      • pypln.apps.extractor: New Document processor
      • pypln.apps.frequency_counter: This script Created on 28/09/11 by flavio
      • pypln.apps.parser
      • pypln.apps.postagger: Part of speech tagger based on NLTK Created on 27/09/11 by flavio
    • pypln.logger: Define logger infrastructure for the cluster
    • pypln.servers
      • pypln.servers.baseapp: Basic application setup classes
      • pypln.servers.ventilator: Task ventilator Binds PUSH socket to tcp://localhost:5557 Sends batch of tasks to workers via that socket Adapted from 0MQ tutorial ("The Guide")
    • pypln.sinks
      • pypln.sinks.base: Base classes for sinks
      • pypln.sinks.dummy_sink: Task sink Binds PULL socket to tcp://localhost:5558 Collects results from workers via that socket
      • pypln.sinks.mongo_insert_sink: Sink to save documents in a MongoDb Collection
      • pypln.sinks.mongo_update_sink: Created on 28/09/11 by flavio
      • pypln.sinks.raw_document_sink: Document sink Binds PULL socket to tcp://localhost:5558 Collects text from workers via that socket and stores in the Document database
    • pypln.stores: This package contains code defining persistent stores which can be used by the sinks to save data to, and by the servers to fetch data for further processing
      • pypln.stores.filestor: Configures an interface for accessing files in MongoDb's GridFS
      • pypln.stores.mongocorpus: This module implements a MongoDB backed NLTK Corpus reader.
      • pypln.stores.mongostore: Define a Mongo Document database for storing raw text documents and metadata
    • pypln.test_messaging: Test module for testing worker spawning and killing workers and sinks
    • pypln.test_worker_processes: This testing module contains tests of the process methods of workers These methods should be called directly, with a message, to avoid the need of starting ventilators and worker processes communicating through ZMQ
    • pypln.testing: Please describe this module.
      • pypln.testing.zmqtesting: This module contains various functions and classes to support testing. Lifted from https://bitbucket.org/ssc/pyzmq-article/src/tip/example_app/test/support.py
    • pypln.workers
      • pypln.workers.base: Base classes for workers
      • pypln.workers.dummy_worker: Task worker Connects PULL socket to tcp://localhost:5557 Collects workloads from ventilator via that socket Connects PUSH socket to tcp://localhost:5558 Sends results to sink via that socket
      • pypln.workers.enc_detect_worker: Character encoding detection worker
      • pypln.workers.highlighter_worker: This worker highlights words on a document using html and CSS.
      • pypln.workers.mime_detect_worker: Mimetype detection worker
      • pypln.workers.palavras_worker: class: palavras_worker connect worker to palavras (bick), and return annotation Author: Agnaldo L Martins (agnaldo@toptc.com.br)
      • pypln.workers.parser_worker: Text parsing worker
      • pypln.workers.pdfconv_worker: PDF to text conversion worker
      • pypln.workers.tagger_worker: This worker does Part of speech tagging.
